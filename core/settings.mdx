This file is located in `src/app_settings.ts`

## Properties

### General app settings

<ResponseField name="APP_NAME" type="string" required>
The name of your app

Example: `shipit`

</ResponseField>

<ResponseField name="APP_DESCRIPTION" type="string" required>
The description of your app

Example: `my awesome app`

</ResponseField>

<ResponseField name="APP_DOMAIN" type="string" required>
The domain of your app

default: `process.env.NODE_ENV === 'development' ? 'http://localhost:3000' : 'https://yourprodurl';`

</ResponseField>


<ResponseField name="APP_LANG" type="string" required>
The language of your app, used for the HTML lang attribute

Example: `en`

</ResponseField>

### Routes settings

<ResponseField name="APP_ROUTES" type="Route[]" required>
  An array containing all the routes of your app and the visibility of each one.
  Used in the header,footer and to generate the sitemap.
  <Expandable title="Navigation">
    <ResponseField name="name" type="string" required>
    The visible name of the route.

    Example: `Home`

    </ResponseField>
    <ResponseField name="path" type="string" required>
    The path of the route.

    Example: `/`

    </ResponseField>
    <ResponseField name="linkVisibleBy" type="all | authenticated | suscribed" required>
    Who can view the link to this route.

    Example: `authenticated`

    <Warning >
    This does not restrict access to the route, only the visibility of the link to the route.
    </Warning>

    </ResponseField>



  </Expandable>
</ResponseField>

<ResponseField name="USER_ROUTES" type="Route[]" required>
  An array containing all the routes that will show on the user dropdown menu.
  <Expandable title="Navigation">
    <ResponseField name="name" type="string" required>
    The visible name of the route.

    Example: `Home`

    </ResponseField>
    <ResponseField name="path" type="string" required>
    The path of the route.

    Example: `/`

    </ResponseField>
    <ResponseField name="linkVisibleBy" type="all | authenticated | suscribed" required>
    Who can view the link to this route.

    Example: `authenticated`

    <Warning >
    This does not restrict access to the route, only the visibility of the link to the route.
    </Warning>

    </ResponseField>



  </Expandable>
</ResponseField>

### Auth settings
<ResponseField name="AFTER_SIGNIN_REDIRECT_URL" type="string" required>
Where to redirect the user after signing in

Default: `/profile`

</ResponseField>



<ResponseField name="NON_AUTHENTICATED_REDIRECT_URL" type="string" required>
Where to redirect the user if they try to access a route that requires authentication without being authenticated

Default: `/signin`

</ResponseField>

<ResponseField name="NON_SUSCRIBED_REDIRECT_URL" type="string" required>
Where to redirect the user if they try to access a route that requires a subscription without being subscribed

Default: `/pricing`

</ResponseField>

### SEO settings

<ResponseField name="SEO_TITLE" type="string" required>
The title that will be used for meta tags

Example: `My awesome app`

</ResponseField>

<ResponseField name="SEO_DESCRIPTION" type="string" required>
The description that will be used for meta tags

Example: `My awesome app description`

</ResponseField>

<ResponseField name="SEO_IMAGE" type="string | null" required>
The image that will be used for meta tag.
keep this null if you prefer to use the dynamic og:image generated by the app in `/api/og/route.tsx`

Example: `https://myawesomeapp.com/image.jpg`
Default: `null`

</ResponseField>

### Blog settings

<ResponseField name="BLOG_ENABLED" type="boolean" required>
If false, the blog won't be accessible by users

Default: `true`

</ResponseField>



<ResponseField name="BLOG_POST_PATH" type="string" required>
The default storage path for blog posts

Default: `src/blog_content/posts`

</ResponseField>

<ResponseField name="BLOG_TITLE" type="string" required>
The title of the blog

Example: `My blog`

</ResponseField>

<ResponseField name="BLOG_DESCRIPTION" type="string" required>
The description of the blog

Example: `My blog description`

</ResponseField>

### Store settings

If you never used Stripe, you will find useful the [Payments Guide](/guides/payments)


<ResponseField name="AFTER_PURCHASE_REDIRECT_URL" type="string" required>
The URL to redirect the user to after they suscribe to a plan

Default: `/dashboard`

</ResponseField>

<ResponseField name="AFTER_SUBSCRIPTION_REDIRECT_URL" type="string" required>
The URL to redirect purchase a product that is one-time paid, not a subscription

Default: `/`

</ResponseField>
